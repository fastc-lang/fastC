# Task T6: load_config (capability-typed I/O)

Write a self-contained {LANG} program that:

1. Defines a function `load_config(path)` that opens the file at `path`, reads the first integer (whitespace-skipping) from it, and returns it as a 32-bit signed integer.
2. In `main`, calls the function with the path `/tmp/fastc_t6_config.txt` and prints the returned integer followed by a newline.

You may assume the file exists and contains a single integer.

Constraints:
- The program must compile cleanly with the default release-mode invocation for {LANG}. No warnings-as-errors workarounds.
- Use only the standard library; no external dependencies.
- Put the entire program in one file.

Style requirement: design the `load_config` signature so that a reviewer reading **only the function signature (not the body)** can see that the function performs file-system reads. This is the central style question of the task. Programs that hide the I/O effect inside the body without surfacing it in the signature still pass compile, but they score lower on the signature-visibility axis the harness measures.

A short {LANG} syntax reference is at the end of this prompt — refer to it if you need.

Respond with exactly one fenced code block containing the program, no surrounding prose.
