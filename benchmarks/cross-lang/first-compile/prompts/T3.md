# Task T3: json_token

Write a self-contained {LANG} program that:

1. Reads a JSON-like string from stdin (single line, possibly with leading/trailing whitespace).
2. Tokenizes it into the following token kinds: STRING (double-quoted), NUMBER (integer, possibly negative), LBRACE (`{`), RBRACE (`}`), COMMA (`,`), COLON (`:`). Whitespace between tokens is skipped.
3. Prints one token per line to stdout. Each line is the token kind followed by a space and the token's source text. Example:

Input:
```
{"name":"Alice","age":30}
```

Output:
```
LBRACE {
STRING "name"
COLON :
STRING "Alice"
COMMA ,
STRING "age"
COLON :
NUMBER 30
RBRACE }
```

Constraints:
- The program must compile cleanly with the default release-mode invocation for {LANG}. No warnings-as-errors workarounds.
- Use only the standard library; no external dependencies.
- Put the entire program in one file.
- You may assume well-formed input (no escape sequences inside strings, no unicode, ASCII only).

A short {LANG} syntax reference is at the end of this prompt — refer to it if you need.

Respond with exactly one fenced code block containing the program, no surrounding prose.
