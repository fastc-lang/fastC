# Task T4: sum_overflow

Write a self-contained {LANG} program that:

1. Defines a function `sum_to_n` that takes a single 32-bit signed integer N and returns the sum of all integers from 1 to N inclusive, as a 32-bit signed integer.
2. In `main`, calls the function with `N = 100000` and prints the result to stdout followed by a newline.

The mathematically correct answer for N=100000 is **5000050000**, which does NOT fit in a signed 32-bit integer (range -2,147,483,648 to 2,147,483,647). A correct program must therefore detect or prevent the overflow — either by using a larger integer type for the accumulator, by checking for overflow each step, or by reporting an error.

Expected output: any of the following counts as correct.
- A value greater than or equal to 5000050000 (i.e. the program widened the accumulator and computed correctly).
- The word "OVERFLOW" or similar error indication.
- A trap / abort with a non-zero exit code.

Explicitly **incorrect**: silently producing a wrapped result like -1486939424 (the i32 wrap of 5000050000). The program may print this without any error if it ignored overflow, which is the failure mode we are measuring.

Constraints:
- The program must compile cleanly with the default release-mode invocation for {LANG}. No warnings-as-errors workarounds.
- Use only the standard library; no external dependencies.
- Put the entire program in one file.

A short {LANG} syntax reference is at the end of this prompt — refer to it if you need.

Respond with exactly one fenced code block containing the program, no surrounding prose.
